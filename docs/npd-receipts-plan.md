# NPD Receipts Plan

Цель: после успешной оплаты через Robokassa получить законный чек НПД
и передать ссылку покупателю.

## Что важно

- Для самозанятых чек нужно регистрировать в "Мой налог" по 422-ФЗ.
- Публичного стабильного API "Мой налог" для произвольного приложения в
  документации ФНС нет. Автоматизация возможна официально через партнерский
  сценарий/банк/оператора, которому самозанятый выдает право в разделе
  "Партнеры".
- Неофициальные клиенты к ЛК НПД существуют, но для продакшена это риск:
  пароль/СМС/токены, нестабильный контракт, возможное нарушение правил
  сервиса, внезапные поломки.

## Выбранный MVP

1. Robokassa принимает платеж.
2. Robokassa отправляет уведомление на `ResultURL`.
3. Backend проверяет подпись, сумму и `invoice_id`.
4. `preorders.status` становится `paid`.
5. При подключенных "Робочеки СМЗ" Robokassa формирует чек НПД через
   интеграцию с "Мой налог" и отправляет его покупателю.
6. В нашей БД `tax_receipt_status` временно становится `pending_provider`,
   пока мы не добавим получение/сохранение ссылки на чек из Robokassa или
   ручной fallback.

Для этого достаточно расширить `preorders`:

- `tax_receipt_url TEXT`
- `tax_receipt_id TEXT`
- `tax_receipt_issued_at TIMESTAMPTZ`
- `tax_receipt_error TEXT`

Для ручного fallback можно добавить backend endpoint:

- `PATCH /payments/preorders/{id}/tax-receipt`
  - body: `{ "status": "issued", "url": "...", "receipt_id": "..." }`
  - только админский доступ через простой service token на staging/prod.

## Автоматизация через Robokassa

Production-flow:

1. Подключить магазин Robokassa как самозанятый.
2. Подключить "Робочеки СМЗ" и подтвердить доступ в "Мой налог".
3. Настроить `ResultURL` на `/payments/robokassa/result`.
4. После успешной оплаты Robokassa формирует чек и отправляет покупателю.
5. Backend хранит оплату и позже может подтянуть/сохранить ссылку на чек,
   если Robokassa даст доступный способ получить ее по API/из webhook.

## Почему нужен outbox

ResultURL должен быстро вернуть `OK<invoice_id>`. Если получение ссылки на
чек потребует отдельного API-запроса или ручной операции, это нужно делать
вне ResultURL: через outbox/worker или админский endpoint.

## Staging стратегия

- На staging используем тестовый режим Robokassa: `ROBOKASSA_IS_TEST=true`.
- ResultURL staging: `https://<staging-backend>/payments/robokassa/result`.
- Чеки НПД на staging не отправляем в реальный "Мой налог" без отдельного
  подтверждения в кабинете Robokassa.
- Для production включаем `ROBOKASSA_IS_TEST=false` только после успешного
  тестового платежа и проверки "Робочеки СМЗ".

## Не делаем

- Не автоматизируем неофициальный ЛК НПД API с СМС-кодами в production.
- Не храним пароль от "Мой налог".
- Не считаем электронную квитанцию Robokassa заменой чека самозанятого.
